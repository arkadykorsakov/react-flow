# React Flow - сценарии

Веб-приложение для просмотра и редактирования сценариев в виде графа (узлы и связи) на базе React Flow.

## Как запустить

Требуется **Node.js** с npm (рекомендуется актуальный LTS).

```bash
npm install
npm run dev
```

Откройте в браузере адрес, который выведет Vite (обычно `http://localhost:5173`).

Дополнительные команды:

| Команда           | Назначение                          |
| ----------------- | ----------------------------------- |
| `npm run build`   | Проверка TypeScript и сборка в `dist` |
| `npm run preview` | Локальный просмотр production-сборки  |
| `npm run lint`    | ESLint                              |
| `npm run format`  | Prettier (запись)                   |

## Стек

- **React 19**, **TypeScript**
- **Vite** — сборка и dev-сервер
- **React Flow** (`reactflow`) — редактор графа
- **Redux Toolkit** + **react-redux** — состояние списка сценариев, загрузка и сохранение (в т.ч. имитация сети)
- **React Router** — маршрутизация
- **Tailwind CSS v4** (`@tailwindcss/vite`) — стили

## Архитектура

Проект ориентирован на **Feature-Sliced Design (FSD)** — слои с фиксированным направлением зависимостей:

| Слой / папка | Роль |
| ------------ | ---- |
| `app`        | Точка входа, провайдеры (store, router), layout |
| `pages`      | Страницы (композиция виджетов и фич) |
| `widgets`    | Крупные блоки интерфейса (шапка, список сценариев) |
| `features`   | Пользовательские сценарии («редактировать сценарий», модалка) |
| `entities`   | Бизнес-сущность «сценарий»: типы, slice, UI узлов графа, форма редактирования |
| `shared`     | Переиспользуемый UI, хуки, утилиты |
| `mocks`      | Мок-данные для списка сценариев до подключения реального API |

Импорты завязаны на алиасы (`app`, `entities`, `features`, `shared`, `mocks` и т.д.) — см. `vite.config.ts` и `tsconfig.app.json`.

**Данные:** список сценариев приходит из моков с искусственной задержкой; сохранение редактирования обрабатывается через `createAsyncThunk` с задержкой и обновлением Redux — как задел под реальный HTTP.

## Отчёт об использовании ИИ

В процессе разработки использовалась IDE Cursor с интеграцией инструментов искусственного интеллекта.

Искусственный интеллект применялся для решения следующих задач:

#### 1. Разработка пользовательского интерфейса
- Верстка компонентов интерфейса
- Генерация и доработка UI-кода

#### 2. Настройка проекта
- Подключение библиотек
- Глобальная настройка зависимостей
- Настройка роутинга
- Настройка хранилища состояния (store)

#### 3. Архитектурные решения
- Использование режима Ask для получения рекомендаций
- Определение структуры проекта
- Консультации по распределению компонентов по слоям (например, entities и features)
